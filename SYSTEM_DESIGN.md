# 个性化资源生成与学习多智能体系统 — 系统设计说明书

## 1. 需求分析

### 1.1 项目背景
基于大语言模型（LLM）构建多智能体协同系统，实现对学生个性化学习画像的动态建模，并据此生成五种个性化学习资源，规划学习路径，提供智能辅导和学习效果评估。

### 1.2 功能需求
| 功能模块 | 描述 |
|---------|------|
| 动态学生画像 | 从对话中提取6个维度画像（知识基础、学习风格、薄弱点、兴趣、节奏、交互摘要），随学随新 |
| 意图识别 | 区分问候/资源生成/普通对话/辅导/评估五类意图 |
| 多智能体资源生成 | 5个智能体协作生成5种资源（规划→内容→多模态→案例→合并） |
| 路径规划与资源推送 | 基于画像+课程知识库生成学习路径，推送匹配资源 |
| 智能辅导 | 针对学生疑问提供概念解释+图解+视频脚本建议 |
| 效果评估 | 分析练习记录，生成评估报告，动态调整推荐策略 |
| 内容安全 | 敏感词过滤和输出安全检测 |

### 1.3 非功能需求
- 流式输出：大模型响应实时逐字推送
- 响应时间：单次交互控制在合理范围内，进度状态实时反馈
- 跨平台：同时支持控制台和 Web 界面
- 透明度：标注所有使用的开源工具和AI工具来源

### 2.1 智能体节点说明

| 智能体角色 | 节点名称 | 输入 | 输出 | 功能描述 |
|-----------|---------|------|------|---------|
| 画像分析智能体 | profile_agent | 最近对话消息 | 6维画像JSON | 从对话中自动提取学生画像，每次对话后更新 |
| 意图识别智能体 | classify_intent | 最后一条消息 | 意图标签 | 分类用户意图至5类 |
| 问候智能体 | greeting_node | - | 问候语 | 简单问候响应 |
| 普通对话智能体 | chat_node | 对话历史+画像+课程上下文 | AI回复+资源推送 | 根据画像调整回答风格，附加学习路径推送 |
| 智能辅导智能体 | tutor_node | 问题+画像+课程上下文 | 文字+图解+视频脚本 | 提供多模态答疑解惑 |
| 效果评估智能体 | evaluation_node | 对话历史+画像 | 评估报告+调整建议 | 分析学习效果，动态调整推荐 |
| **资源规划智能体** | resource_planner_agent | 画像+课程知识库+问题 | JSON规划方案 | 规划资源内容、顺序和教学方法 |
| **内容生成智能体** | content_generator_agent | 规划+画像+课程 | 讲解文档+练习题 | 生成教学文档和练习题目 |
| **多模态设计智能体** | multimodal_generator_agent | 规划+画像+课程 | 思维导图+视频脚本 | 设计可视化内容和视频脚本 |
| **案例生成智能体** | case_generator_agent | 规划+画像+课程 | 实操案例+代码 | 生成实践项目案例 |
| **资源合并智能体** | resource_merger_agent | 各智能体输出 | 完整5种资源 | 校验完整性，合并输出 |
| **路径规划智能体** | plan_node | 画像+课程知识库 | 3-5步学习路径 | 规划个性化步骤并推荐资源类型 |

### 2.2 状态定义 (AgentState)

```python
{
    "messages": List[BaseMessage],         # 对话历史
    "user_intent": str,                     # 意图标签
    "profile": Optional[StudentProfile],    # 6维画像
    "learning_plan": Optional[List[str]],   # 路径步骤列表
    "course_context": Optional[str],        # 课程知识库上下文
    "resource_plan": Optional[str]          # 资源规划方案
}
```

## 3. 课程知识库

系统内置《数据结构与算法》完整课程知识库（course_knowledge_base.json），包含：

| 内容 | 数量 |
|------|------|
| 章节目录 | 8章（绪论、线性表、栈与队列、串与数组、树与二叉树、图、查找、排序） |
| 知识点 | 60+（含难度等级、先修关系） |
| 练习题 | 30+（选择题+编程题） |
| 资源模板 | 5种（对应5种资源类型的结构模板） |
| 知识图谱 | 知识点关联关系+学习路径推荐 |

系统自动根据用户输入匹配知识库中的相关章节和知识点，作为智能体的上下文参考。

## 4. 关键技术

### 4.1 LangGraph 多智能体框架
- 使用 `StateGraph` 定义有向图工作流
- 节点间通过共享状态 `AgentState` 通信
- `add_messages` 注解自动管理消息历史
- 条件路由根据意图标签分发到不同节点
- 资源生成阶段使用5个智能体协作（规划→内容→多模态→案例→合并）

### 4.2 讯飞星火 API（科大讯飞）
- 模型：Spark X1（支持在线搜索）
- 流式模式：`stream=True`
- 网络搜索：内置 `web_search` 工具
- 思维链输出：`reasoning_content` 字段

### 4.3 SSE 实时推送
- `sse-starlette` 实现 Server-Sent Events
- 前端 `ReadableStream` 逐块读取
- 支持断线检测 `request.is_disconnected()`

### 4.4 课程知识库匹配
- 关键词匹配算法，自动关联用户提问与课程知识点
- 匹配结果作为智能体提示词的上下文注入

### 4.5 安全机制
- 敏感词列表检查（输入/输出双向过滤）
- 输出内容替换为安全提示
- API 请求超时设置（120秒）

## 5. 创新实践

1. **动态画像融合**：每次对话更新画像，旧画像与新提取信息合并，支持持续进化
2. **多智能体资源协作**：资源生成阶段拆分为5个专业智能体，各司其职后合并输出
3. **课程知识库驱动**：内置完整高校课程知识库，智能匹配知识点上下文
4. **多模态资源**：不仅生成文档和代码，还生成思维导图文本结构和视频分镜脚本
5. **画像驱动的精准推送**：根据知识基础、学习风格、薄弱点三方面匹配资源类型
6. **卡片化Web展示**：前端自动检测资源标题，用彩色卡片分组展示
